PUP

Motivation

A spec file that describes the structure of a bundle, it's ingrediants and extra metadata required for CTT-Service swapping the plist on the fly based on PUP prediction as well as a contract between CTT & PUP.

Prediction Key

each resource in a bundle can get one of the possible predecitions: None: Not targeted by PUP --> will be marked by "L" Monitor: a mode used for PUP for prediction validation --> will be marked by "L" Short: a mode that indicate a good health and the short list can be used --> will be makred by "S"

Spec File Main Elements

  1. Bundle ID: bundle Name and metadata required for humans and tools.

  2. Bundle Structure: includes the resources and the bundles lists.

  3. Bundle Permutations: a mapping from PredictionKey

BundleName

BundleName, associated plist and hosting instance share the same naming structure.

MCTT _ [TAG] _ [BUNDLEALIAS] _ [CTFFREQ]

example: mctt_0p0_b17_400
